# Case Study: Biodiversity Dataset Cleaning

This case study is grounded in the sample cleanup assets already stored in the
repo. It should be treated as real sample work, not as a named client case
study.

## Starting Point

The source dataset contained 27 biodiversity survey rows and needed cleanup
before it could be trusted for analysis, mapping, or reporting.

## Problems Addressed

- inconsistent date formatting
- species-name variation
- duplicate records
- coordinate issues
- habitat-label inconsistency
- missing traceability around corrections

## Work Performed

The cleanup pass produced:

- date normalization
- species-name harmonization
- duplicate removal
- GPS correction where possible
- habitat standardization
- validation notes and correction tracking

## Output

The final dataset was reduced to 25 consistent records and organized into a
more analysis-ready, GIS-compatible, and reporting-friendly structure.

## Why This Matters

For many biodiversity teams, the bottleneck is not field collection alone. It
is the last stage before analysis or donor reporting, where messy tables delay
real use. A bounded cleanup pass can remove that friction without forcing a
large consulting process.

## What This Case Study Proves

- the service is grounded in a real cleanup workflow
- corrections can be documented rather than hidden
- the founder can produce traceable outputs instead of vague promises

## What It Does Not Claim

- no named client endorsement
- no testimonial
- no long-term contract claim
- no claim that every dataset can be fixed the same way
