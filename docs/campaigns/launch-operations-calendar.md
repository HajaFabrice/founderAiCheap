## Weekly operating rhythm

### Sunday batch block

Use Sunday as the non-negotiable prep block.

- review the next four scheduled posts
- copy the correct body from the post bank
- confirm the correct first-comment route in the execution sheet
- confirm the matching visual asset
- preload any first-comment text in a notes file
- confirm you can actually be online during all post windows

### Monday engagement block

- leave 5 to 10 substantive comments on relevant accounts
- send a small number of high-quality connection requests
- reply to any leftover DMs from the prior week

### Tuesday to Friday posting block

For each live post:

1. publish only when you can stay online for the first hour
2. add the first comment within 5 minutes
3. reply to comments quickly and thoughtfully
4. log any strong reactions, recurring questions, or DM signals

### Saturday review block

- log the best post by comments
- log the best post by profile visits or clicks if available
- note which CTA route produced the strongest intent
- identify one thing to keep and one thing to adjust next week

## What the external roadmap said that we are not following literally

These were useful in spirit but not correct for the current repo state:

- not forcing Kit before any public launch
- not requiring all 10 products to go live before the campaign runs
- not sending cold LinkedIn traffic directly to checkout
- not treating Shopify as the permanent center of the system

## Done definition

This calendar is being used correctly when:

- every week has four prepared posts
- every post has the right link and visual before publish time
- no cold post sends people straight to checkout
- launch decisions are made from one operator pack, not from memory
