# Launch Readiness Checklist

Purpose: one realistic pre-launch checklist for the current OPLURIX system.

This is adapted from the external operational roadmap, but filtered through the
current repo truth:

- one live product
- Payhip checkout
- Shopify as temporary storefront
- repo site as authority + funnel layer
- checklist funnel exists
- Kit is still optional / not yet fully connected

## 1. Before The First New Launch Push

All of this should be true before you restart a visible launch campaign.

### Profile and public identity

- [ ] LinkedIn field-facing profile photo is current
- [ ] LinkedIn banner is current and aligned with OPLURIX
- [ ] headline and About are current
- [ ] Featured section order is intentional:
  1. checklist or homepage
  2. live product
  3. ATBC research page
- [ ] Facebook page uses current cover + profile image

### Website and funnel

- [ ] repo site is deployed publicly
- [ ] homepage works on mobile
- [ ] checklist EN page works
- [ ] checklist FR page works
- [ ] printable EN checklist works
- [ ] printable FR checklist works
- [ ] thank-you flow works for checklist submissions
- [ ] campaign preview images refresh correctly

### Email/follow-up mode

Pick one mode before launch:

- [ ] Mode A: manual / no Kit yet
- [ ] Mode B: Kit connected

If Mode A:

- [ ] manual checklist follow-up process is documented
- [ ] lead review cadence is decided

If Mode B:

- [ ] Kit forms exist
- [ ] Kit tags exist
- [ ] Kit delivery email exists
- [ ] welcome sequence starts correctly

### Offer truth

- [ ] Product 1 description is current on Payhip
- [ ] Product 1 guarantee text is current
- [ ] repo product page matches Payhip
- [ ] Shopify copy does not imply more live products than reality
- [ ] Products 2 and 4 are labeled as next-release, not fully live
- [ ] Product 3 is labeled as held for refresh, not part of the clean next-release set

### Routing truth

- [ ] cold traffic goes to checklist or homepage
- [ ] warm traffic goes to product page
- [ ] hot traffic goes to direct checkout
- [ ] research-support traffic goes to ATBC page

## 2. Non-Negotiable Launch Rules

- [ ] Do not publish a post if you cannot be present in the first hour
- [ ] Do not put links in the body of LinkedIn posts
- [ ] Put the first comment up within 5 minutes
- [ ] Reply within 60 minutes when possible
- [ ] Do not hard-sell Product 1 under every post
- [ ] Do not present Products 2 to 10 as all live now
- [ ] Do not promise automated delivery if the current mode is still manual

## 3. What To Delay On Purpose

Do not block the launch waiting for these:

- full Netlify migration
- Shopify checkout migration
- Products 5 to 10 activation
- perfect analytics

## 4. Go / No-Go Decision

Launch is a `GO` only if:

- public pages are live
- checklist route works
- Product 1 route works
- follow-up mode is known
- public messaging is aligned

If one of those fails, fix it before posting.
