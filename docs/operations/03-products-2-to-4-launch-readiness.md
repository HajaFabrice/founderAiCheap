# Products 2 To 4 Launch Readiness

Purpose: move Products 2 to 4 from "good-looking pages" to actual sellable releases.

## Launch order

Recommended:

1. Product 2 - Training-to-Quiz Generator
2. Product 3 - Gear & Equipment Concierge
3. Product 4 - Biodiversity Pitch Deck Builder

## Why this order

- Product 2 is practical and low-friction.
- Product 3 is strong utility but benefits from precise framing.
- Product 4 is higher-value and needs stronger sales confidence.

## Readiness criteria for each product

Each product must pass all five gates before activation:

### Gate 1. Public message gate

- page headline is clear
- promise is specific
- status is honest
- price is confirmed

### Gate 2. Delivery gate

- English delivery pack exists
- French delivery pack exists if bilingual delivery is intended
- files are bounded and readable

### Gate 3. Support boundary gate

- support limit is explicit
- no hidden custom-work promise
- fulfillment method is known

### Gate 4. Checkout gate

- exact checkout destination is chosen
- description is ready
- guarantee position is set if used

### Gate 5. Operator gate

- fulfillment file naming is decided
- log template is ready
- language version decision is easy

## Product-specific notes

### Product 2

Strongest first use:

- NGO trainers
- field team leaders
- workshop facilitators

Key risk:

- sounding too generic if not anchored in real SOP use

### Product 3

Strongest first use:

- early-career field biologists
- grant writers
- NGO field managers

Key risk:

- implied price-tracking promise that is too broad

### Product 4

Strongest first use:

- conservation scientists
- NGO donor-facing staff
- grant applicants

Key risk:

- high perceived value requires higher clarity and trust

## Activation checklist

Before turning any one of them live:

1. confirm price
2. confirm status wording
3. confirm checkout path
4. confirm delivery asset
5. confirm fulfillment log row can be used
6. confirm bilingual state

## Done means

- the product can actually be sold
- it can be delivered without confusion
- its page is no longer ahead of its operations
