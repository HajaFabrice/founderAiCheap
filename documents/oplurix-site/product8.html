Section 1

The Core Promise

Thermal drone surveys produce data at a rate that quickly overwhelms manual processing. A single 25-minute thermal flight at Ihofa Forest generates between 300 and 800 image files, depending on flight speed and trigger interval. Each image must be reviewed, detections must be validated, false positives must be removed, GPS coordinates must be extracted and verified, and the results must be entered into a structured detection dataset. Multiply this by a week of field surveys and you have a data processing challenge that can take longer than the survey itself — if you do not have a system.

This guide gives you that system. It is built on the actual workflow used for the thermal drone lemur surveys presented at IPS Congress 2025 — the proof-of-concept that underpins the ERIS methodology. It covers the full pipeline from raw .RJPEG or .TIFF thermal output to a validated, GPS-tagged, confidence-flagged wildlife detection dataset that is directly compatible with the OPLURIX Biodiversity Data Harmonizer and the ERIS scoring framework.

Section 2

10 Real Pain Points

01
You cannot tell a warm animal from a warm rock.

Thermal signatures are temperature differentials, not species identifications. Sun-warmed rocks, decomposing vegetation, and reflective surfaces all produce warm signatures that look like animals in thermal imagery. Without a systematic false positive protocol, your detection count is inflated and your dataset is scientifically unusable.

02
The same individual appears in multiple consecutive frames.

A thermal drone flying at 5 m/s with a 1-second image trigger interval takes a new image every 5 metres. A 2-metre animal can appear in 3–5 consecutive frames. Without a frame-merging protocol, you count one animal five times.

03
You cannot identify species from thermal imagery alone.

Thermal cameras detect heat, not morphology. A small lemur and a large gecko at the same distance produce different-sized signatures — but size alone cannot reliably distinguish species in dense canopy. Species classification requires a combination of thermal signature shape, size estimation, behaviour, habitat context, and cross-validation with concurrent audio or RGB data.

04
Dense canopy blocks thermal signatures completely.

Under a closed forest canopy, the thermal camera sees canopy surface temperature — not the animals underneath. Detection rates in closed canopy are dramatically lower than in open canopy or edge habitats, but this difference is a detection artefact, not a real ecological signal. It must be accounted for in any analysis.

05
GPS metadata in thermal image files is often inaccurate or missing.

Consumer-grade drones (including the Mavic 3T) embed GPS coordinates in image EXIF data from the aircraft's onboard GPS — not RTK-grade positioning. The typical accuracy is ±2–5 metres in open sky, degrading to ±10–20 metres in forested terrain where satellite acquisition is poor. Understanding and documenting this uncertainty is essential for spatial analysis.

06
You have no system for consistent image review across multiple observers.

When two people review the same thermal image sequence, they will flag different detections. Without a standardised review protocol and inter-observer calibration, your detection dataset reflects the reviewer as much as the actual wildlife.

07
The thermal image file format is not directly importable into standard analysis tools.

DJI thermal cameras produce .RJPEG files — a proprietary format that contains both a standard JPEG preview and raw radiometric temperature data. Standard image viewers show the JPEG. The radiometric data, which is required for temperature-based analysis, requires specific extraction tools. Most researchers never access the raw temperature data at all.

08
Time of survey profoundly affects detection probability — but nobody documents it systematically.

Thermal contrast between an endothermic animal and the environment is highest in the hours before dawn, when the environment is coldest. At midday in a tropical forest, thermal contrast may be so low that detection is near-impossible. This is not a equipment failure — it is a survey design issue. But without systematic documentation of survey time and ambient temperature, the detection probability varies dramatically across a dataset with no way to model it.

09
YOLO models trained on other ecosystems perform poorly on your species.

A YOLO model trained on African savanna wildlife will not reliably detect Malagasy forest lemurs in dense canopy thermal imagery. Models must be trained or fine-tuned on imagery from your specific ecosystem, your specific species, and your specific flight parameters. This guide explains what you need to build and validate a context-specific model.

10
You cannot connect thermal detections to the ERIS scoring framework without a structured output format.

The ERIS ecological integrity scoring system requires detection data in a specific schema — with confidence flags, GPS coordinates, species classification confidence, canopy cover estimates, and effort metadata all attached to each record. Raw thermal imagery does not produce this format. This guide ends with the exact output schema needed for ERIS integration.

Section 4

The 7-Step Thermal Processing Pipeline

From raw image files to validated detection records. Every step has a documented output. Steps 1–3 are image-level work. Steps 4–7 build the scientific dataset.

1

FILE ORGANISATION AND METADATA EXTRACTION

Before reviewing a single image, organise the files and extract the metadata. This step defines the structure every subsequent step works within.

File organisation protocol
  • One folder per flight. Naming: [SiteCode]_[Date]_[FlightNum] — e.g., IHOFA_20260315_F01
  • Do not rename individual image files. The original filename contains the sequence number used for consecutive-frame merging.
  • Copy all files to working storage before processing. Never work from the original memory card.
  • For each flight folder: create a _metadata.csv file with one row per image, extracting: filename, GPS latitude, GPS longitude, GPS altitude, image timestamp, camera model, thermal palette setting.

Use the Python EXIF extraction template in Section 9 (Prompt 10) to batch-extract metadata from all files in a flight folder. The exiftool command-line tool is the most reliable option for DJI thermal files.

Critical: RJPEG vs. TIFF file handling

DJI thermal cameras (Mavic 3T, H30T, XT2) produce .RJPEG files. These contain a standard JPEG preview (visible to any image viewer) and embedded radiometric data (temperature per pixel). For wildlife detection, the JPEG preview is usually sufficient. For temperature-based analysis (habitat thermal mapping, thermal habitat quality scoring for ERIS), the radiometric data must be extracted separately using DJI's dji_thermal_sdk or the open-source thermal_parser Python library.

Never delete RJPEG files after extracting the JPEG preview — the radiometric data is embedded in the original file and cannot be recovered once the file is deleted.

Output: Organised flight folders + metadata CSV per flight.

2

THERMAL PALETTE NORMALISATION

Before reviewing images for detections, ensure all images in a flight use the same visual presentation. Mixed palettes (some images in Ironbow, some in Rainbow) make cross-image comparison impossible and dramatically increase reviewer error rates.

Recommended palette for wildlife detection

Ironbow (Iron/Hot): Warm objects appear orange-white, cold objects appear dark blue-black. High contrast between endothermic animals and cool vegetation. This is the standard palette for wildlife detection in forested environments.

Cold (dark)→ Ironbow palette →Hot (white/yellow)

For habitat mapping: Rainbow palette provides more gradation across the temperature range. Use for thermal heterogeneity analysis, not for detection review.

Output: All images in consistent Ironbow palette. Note the temperature range displayed (e.g., 15°C–40°C) in the flight metadata — this affects how detections appear.

3

INITIAL DETECTION REVIEW — FLAGGING

Review every image in the flight sequence and flag images containing candidate detections. Do not attempt species identification at this stage. You are answering one question only: "Is there a thermal signature in this image that warrants closer review?"

Detection flagging criteria
  • A blob-shaped warm signature distinctly warmer than the surrounding canopy surface — not a gradual warm area
  • A signature with a shape consistent with a vertebrate body (elongated, roughly symmetrical, with a definable edge)
  • A signature that changes position across consecutive frames (moving animals)
  • A signature that appears and disappears across frames (animal emerging from and returning to canopy cover)

Record the image filename of every flagged frame. The fastest review method is fast-scroll through images at 4–6 frames per second, pausing to flag anything that meets the criteria. A 500-image flight can be reviewed for flagging in 20–30 minutes at this pace.

Output: A flagged image list — filenames of all images containing candidate detections. Typically 5–15% of total images in a forested tropical survey.

4

FALSE POSITIVE REMOVAL

For every flagged image, apply the false positive decision framework. This is the most important step for data quality. A detection dataset with 30% false positives is not usable for any rigorous analysis.

The six most common false positives in tropical forest thermal surveys
  • Sun-warmed rock or boulder: Appears very bright (hotter than expected ambient). Diagnostic: present in same frame across multiple consecutive flights; does not move; edges are sharp geometric shapes.
  • Decomposing vegetation: Mounds of decaying organic matter produce significant heat. Diagnostic: diffuse warm area, irregular edges, no movement, typically on the ground.
  • Termite mounds: Extremely hot signatures. Diagnostic: stationary, very high temperature relative to surroundings, spherical or conical shape, consistent across multiple days.
  • Reflective surfaces: Metal objects, water, and wet leaves can reflect solar radiation and appear warm. Diagnostic: signature too bright (clipping the temperature scale), sharp geometric edges.
  • Thermal bleed from canopy gap: Solar heating of the ground visible through a gap in the canopy creates a warm patch that looks like an animal on the ground. Diagnostic: signature corresponds to a canopy gap visible in simultaneous RGB image; stationary; uniform warming pattern.
  • Aircraft thermal artefacts: Hot pixels or banding from camera sensor artefacts. Diagnostic: same pixel position in every image of the same flight; not correlated with ground features.

Output: Flagged image list with false positives removed. Each removal documented with the false positive category applied.

5

CONSECUTIVE-FRAME MERGING

Merge all consecutive-frame observations of the same individual into a single detection record. This is the most common counting error in thermal drone surveys.

Consecutive-frame merging rules
  • Group flagged images that are sequential in frame number (no gap larger than 3 frames between them)
  • For each group: verify that the signature position is consistent with the drone's flight path (the signature moves through the frame as the aircraft passes)
  • For each group: count one individual, not one image
  • The detection GPS coordinate is taken from the image where the animal is closest to the nadir (directly below the aircraft) — the most accurate GPS estimate for the animal's position
  • If two separate signatures appear in the same frame sequence and maintain consistent separation: count as two individuals
Frame gap rule

A gap of 1–3 frames (1–3 seconds at standard trigger rate) between observations of what appears to be the same animal can occur when the animal briefly moves under canopy between images. A gap larger than 5 frames should be treated as a potential second individual unless the GPS positions are consistent with continuous movement.

Output: One row per unique individual detection. Frame group documented in the source_frames column.

6

SPECIES CLASSIFICATION AND CONFIDENCE ASSIGNMENT

Classify each validated detection to the highest taxonomic level that the data supports with confidence. Do not overstate species identification from thermal data alone.

7

GPS EXTRACTION, VALIDATION, AND EFFORT DOCUMENTATION

Extract GPS coordinates from image EXIF data. Validate against the study area. Document all flight parameters as survey effort covariables.

Effort covariables — record for every flight
  • Flight time: Start and end time (UTC). Used to calculate ambient temperature differential.
  • Flight altitude AGL: Above-ground level in metres. Affects sensor footprint size and detection distance.
  • Flight speed: m/s. Affects consecutive-frame overlap and detection probability.
  • Image trigger interval: Seconds between frames.
  • Ambient temperature: °C at survey time. The single most important covariable for thermal detection probability.
  • Canopy cover: Estimated percentage for each transect pass. Can be derived from RGB imagery or from the thermal image itself (proportion of frame with warm canopy surface).
  • Wind speed: m/s. Above 10 m/s, detection probability drops significantly as animals shelter.
  • GPS accuracy estimate: From EXIF or DJI flight log. Typical Mavic 3T: ±2–5m in open, ±10–20m under canopy.

Output: Detection dataset with validated GPS + flight effort table. Ready for analysis.

Section 5

Thermal Image Interpretation Guide

How to read thermal imagery for wildlife detection. Built from the Mavic 3T field experience at Ihofa Forest, but applicable to any wildlife thermal drone survey in tropical forest.

Understanding What the Camera Sees

A thermal camera measures the radiant temperature of the surface it can see — not the air temperature and not the temperature of objects obscured by other objects. In a forest survey, this means:

  • The camera primarily sees the canopy surface when flying above a closed forest.
  • Animals are only visible when they are at or above the canopy surface, or in a gap in the canopy that allows a sightline to the animal.
  • The canopy surface temperature varies across the image — sun-facing leaves are warmer than shaded leaves, creating a background thermal "texture" that can mask animal signatures.
  • The best detection conditions are early morning (1–3 hours after sunrise) when the canopy surface is still cool from the night and animals are warm relative to their environment.

The Optimal Survey Window

Time of dayDetection conditionsReason
Pre-dawn (04:00–06:00)ExcellentMaximum thermal contrast — animals are warmest relative to near-freezing ambient at altitude
Early morning (06:00–09:00)GoodHigh contrast. Canopy still cool. Most diurnal species active and exposed.

GSD = Ground Sampling Distance. Footprint values are approximate for Mavic 3T FLIR Boson 640 sensor. Altitude choice is a tradeoff between area coverage rate and detection probability for target species.

Canopy Cover and Detection Probability

Canopy cover is the single strongest predictor of detection probability in forested thermal surveys. The relationship is roughly linear: each 10% increase in canopy cover reduces detection probability by approximately 15–25%, depending on canopy structure and species size.

The canopy cover correction — critical for ERIS scoring

A detection rate of 5 animals per km² in open canopy (20% cover) is not comparable to a detection rate of 5 animals per km² in dense canopy (80% cover). The dense canopy site almost certainly has a higher true density — the lower detection rate reflects lower detectability, not lower abundance.

ERIS thermal habitat quality scoring accounts for this by converting raw detection rates into corrected density estimates using canopy cover as a detection probability covariate. The detection data and the canopy cover estimate must both be recorded for this correction to be applied.

Python — EXIF Batch Extraction from DJI Thermal Files

Python · exiftool subprocess · DJI thermal EXIF extraction
import subprocess, csv, os, json def extract_thermal_exif(folder_path: str, output_csv: str) -> None: """Extract GPS and flight metadata from all thermal files in folder.""" cmd = [ 'exiftool', '-r', '-json', '-GPSLatitude#', '-GPSLongitude#', '-GPSAltitude#', '-DateTimeOriginal', '-RelativeAltitude', '-Model', '-FileName', folder_path ] result = subprocess.run(cmd, capture_output=True, text=True) records = json.loads(result.stdout) fields = ['FileName','DateTimeOriginal','GPSLatitude', 'GPSLongitude','GPSAltitude','RelativeAltitude','Model'] with open(output_csv, 'w', newline='') as f: writer = csv.DictWriter(f, fieldnames=fields + ['gps_flag']) writer.writeheader() for r in records: lat = r.get('GPSLatitude') lon = r.get('GPSLongitude') flag = 'VALID' if lat is None or lon is None: flag = 'MISSING_GPS' elif abs(lat) < 0.001 and abs(lon) < 0.001: flag = 'NULL_ISLAND' row = {k: r.get(k, 'NA') for k in fields} row['gps_flag'] = flag writer.writerow(row) print(f"Extracted {len(records)} records → {output_csv}")

Python — Consecutive-Frame Detection Merger

Python · pandas · merge consecutive flagged frames into single detections
import pandas as pd import numpy as np def merge_detections(df: pd.DataFrame, max_gap: int = 3) -> pd.DataFrame: """Group consecutive flagged frames into single detection records.""" flagged = df[df['detection_flagged']].sort_values('frame_number').copy() flagged['group'] = (flagged['frame_number'].diff() > max_gap).cumsum() groups = [] for gid, grp in flagged.groupby('group'): best_idx = ((grp['GPS_lat'] - grp['GPS_lat'].mean())**2 + (grp['GPS_lon'] - grp['GPS_lon'].mean())**2).idxmin() groups.append({ 'detection_id': f"DET_{gid:04d}", 'start_frame': grp['frame_number'].min(), 'end_frame': grp['frame_number'].max(), 'n_frames': len(grp), 'latitude': round(grp.loc[best_idx, 'GPS_lat'], 5), 'longitude': round(grp.loc[best_idx, 'GPS_lon'], 5), 'source_frames': ';'.join(grp['frame_filename'].tolist()) }) return pd.DataFrame(groups)

ERIS Integration — Thermal Habitat Quality Scoring

How thermal drone data contributes to the ERIS ecological integrity score. The thermal survey produces two ERIS indicator inputs: detection-based species richness data and thermal habitat quality metrics.

ERIS Indicator 3 — Thermal Habitat Quality

This indicator characterises the thermal structure of the survey area — the distribution of microhabitat temperatures, canopy cover, and thermal heterogeneity that determine habitat suitability for target species.

ERIS Thermal Habitat Quality — four sub-metrics

3a. Canopy cover index: Mean canopy cover across the survey area from the thermal orthomosaic (%). Higher closed-canopy cover = higher forest integrity score for forest-dependent species.

3b. Thermal heterogeneity index: Variance in canopy surface temperature across the survey area. High heterogeneity = greater microhabitat diversity = higher habitat quality for thermally sensitive species (amphibians, reptiles).

3c. Thermal edge density: Linear metres of thermal contrast edge (forest-to-gap boundaries) per hectare. High edge density can indicate fragmentation or degradation.

3d. Detection-corrected wildlife density: Detections per km² of thermal survey area, corrected for canopy cover using the detection probability model. This connects the thermal detection data to the ERIS species richness indicator.

From Thermal Data to ERIS Score — the Calculation Path

  1. Complete the 7-step processing pipeline → validated detection dataset + flight effort table
  2. Generate the thermal orthomosaic (if multiple overlapping passes) or use individual images for single-pass surveys → thermal habitat raster
  3. Calculate canopy cover, thermal heterogeneity, and edge density from the raster using the R code template (Section 9)
  4. Calculate detection-corrected wildlife density using canopy cover as a detection probability covariate (distance sampling or N-mixture model)
  5. Input all four sub-metrics into the ERIS scoring framework using the Data Harmonizer output schema
Section 12

Three-Check Quality Framework

Check 1 — False Positive Rate Audit

What it tests

What percentage of initial detections were removed as false positives? A rate above 40% indicates systematic problems with survey conditions or review protocol.

Pass criteria: False positive rate documented and below 40%. Each false positive category is logged.

Check 2 — Inter-Observer Agreement

What it tests

If two reviewers independently reviewed the same image subset, how closely do their detections agree? Below 70% agreement indicates that the review protocol needs clarification before full dataset review.

Pass criteria: ≥70% agreement on detection presence/absence. ≥60% agreement on species confidence classification.

Check 3 — GPS Validity and Coverage

What it tests

Are all detection GPS coordinates within the study area? Does the spatial distribution of detections match the flight path? (Detections that cluster far from the flight path indicate GPS error.)

Pass criteria: ≥95% of detections within study area bounding box. Detection spatial distribution consistent with flight path within GPS accuracy estimate.

Section 13

Quick Start — Process Your First Flight

Before You Begin

  • Confirm all thermal image files are copied from the memory card to working storage. Do not process from the card.
  • Record the flight metadata: site, date, time, altitude, ambient temperature, canopy cover estimate, pilot.
  • Install exiftool (command-line) and Python with pandas, pillow.

Hours 1–2 — File Organisation and Extraction

  • Create a flight folder with the correct naming convention.
  • Run the EXIF extraction script (Section 9, Prompt 10). Verify GPS coordinates for a sample of images on a map.
  • Set all images to Ironbow palette with consistent temperature range. Note the range used.

Hours 3–4 — Detection Review and False Positive Removal

  • Fast-scroll through all images at 4–6 fps. Flag images with candidate detections.
  • For each flagged image: apply the false positive decision framework (Section 5). Log every removal.
  • Record: total images, images flagged, false positives removed, false positive rate.

Hour 5 — Consecutive-Frame Merging and Classification

  • Run the consecutive-frame merge script (Section 9) on the validated flagged list.
  • For each merged detection: classify species and assign confidence level using Species Detection Guide (Section 8).
  • Populate all required columns of the detection record schema (Section 10).

Hour 6 — Validation and Documentation

  • Run the three-check QC framework (Section 12). Document results.
  • Use Prompt 12 to generate the flight detection report.
  • Run the ERIS schema converter (Prompt 8) if submitting to ERIS.
  • Back up the processed dataset to two locations.
