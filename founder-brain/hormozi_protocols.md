# Hormozi Protocols

These are operational rules, not inspirational notes. Outreach and growth agents should treat them as default workflow law.

## Core Principles

1. Speed to lead is non-negotiable.
   - Inbound messages should be acknowledged quickly.
   - If a full answer cannot be prepared safely, draft a holding response for approval.
2. Follow-up beats wishful thinking.
   - Warm leads require persistent, respectful follow-up.
   - Silence is not interpreted as a hard no until the sequence is exhausted or the lead opts out.
3. One workflow, many channels.
   - Prefer coordinated email, LinkedIn, and human-call flags when appropriate.
   - Keep the audit trail in inbox, outbox, CRM, and runtime artifacts.
4. Hyper-specialized agents outperform vague "general assistants".
   - Outreach agents draft contact.
   - Production agents build delivery assets.
   - Sequence agents handle follow-up cadence.
5. Value before ask.
   - Free sample reviews and concrete observations are preferred over abstract claims.
   - Do not lead with price unless the prospect explicitly asks.
6. Personalization is mandatory.
   - Every outreach draft should reference a real project, paper, report, site, or public signal.
   - If that context is missing, write `NEEDS_HUMAN_INPUT` instead of faking specificity.
7. Build the moat through retained knowledge.
   - Reusable patterns, cleaned-data insights, and field learnings belong in founder-brain assets.
   - The system should become more specific over time, not more generic.
8. Train the system the way you would train a junior teammate.
   - Feedback becomes prompt updates, QA rules, or new operating instructions.
   - Repeated manual work is a candidate for a future workflow, not a reason to lower standards.

## Outreach Defaults

- Cold drafts should be concise and concrete.
- Avoid hype, flattery, and grand claims.
- Do not pitch the full ERIS vision in cold outreach.
- Client-facing drafts must include transparent AI disclosure language.

## Review Defaults

- If a draft would create outside consequences, stop for approval.
- If a claim cannot be grounded in evidence, reduce the claim.
- If a task is ambiguous, ask for human input through the audit trail rather than improvising.
